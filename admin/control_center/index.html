What this app does

This is a safe local control panel for your DayZ server folder. It reads map status, runs known admin scripts, captures output, and creates snapshots before guarded writes. It does not expose a public web server and it does not accept arbitrary shell commands.

Safe Daily Workflow

  1. Refresh the dashboard.
  2. Check the map you want to work on.
  3. Run Config Drift or Triage Logs if anything looks off.
  4. Create a snapshot before larger changes.
  5. Apply changes, restart the affected map, then smoke test.

Support Report

Generate a public-safe summary you can paste into a support thread or Discord. It lists app version, configs, ports, missions, missing mods, and the latest log state for one map or all maps. Passwords, Steam IDs, and tokens are redacted, and no player, storage, or profile contents are included.

Backups & Restore

Snapshots are local zip backups of public-safe configs (saved under the ignored admin/backups). One is created automatically before every guarded change, and you can make one anytime. Tip: create a snapshot before a scheduled restart or any big edit so you always have a clean point to return to. Restoring overwrites current configs and snapshots the current state first; restart affected maps afterward.

Restoring a snapshot is an Advanced-mode action. Switch to Advanced (top bar) to enable Restore.

Live Admin (RCON)

Connect to a running server's BattlEye RCON to see who's online, broadcast messages, and kick or ban players. Enable RCON once per map, then restart that map so BattlEye loads it — RCON starts answering about 20-30 seconds after the server boots. The password is generated and stored only in the map's private BattlEye folder; it is never tracked or shown.

Keep Servers Up

Auto-Restart on Crash (Keep-Alive)

Turn on keep-alive for a map and the Control Center will relaunch it automatically if it crashes or stops unexpectedly (checked about every 30 seconds, after a short grace period). If a map crash-loops (3 restarts in 10 minutes) it pauses so you can fix the cause first. To take a map offline, turn keep-alive off, then stop it. Runs while this app is open.
Loading keep-alive status...

Scheduled Restarts

Automatically restart each map on an interval, with in-game countdown warnings sent over RCON before each restart (enable RCON for that map first, on the Live Admin tab). Warnings are minutes-before, comma-separated (e.g. 30,15,5,1). Schedules run while this app is open — leave it running on the server box. Restarts stop and relaunch the map; players see the warnings and the world reloads.

Updates (SteamCMD)

Keep the dedicated server and Workshop mods current with SteamCMD. No password is stored here — you log in once in a SteamCMD window (password + Steam Guard), and it caches the session for later updates. Updates open their own console window so you can watch progress. Stop your servers before updating server files.
Checking SteamCMD...

1. Install SteamCMD

Downloads SteamCMD into the ignored local_runtime folder and self-updates. No login needed.

2. Steam login

Saved locally only. You enter your password and Steam Guard in the SteamCMD window, not here.

3. Update

Each opens a SteamCMD window. Stop servers before updating server files; after a mod update, run Sync Workshop mods.
